# Teacher Notes

## Why It Is Here

This assignment makes “grounding” operational instead of rhetorical.

## What A Strong Report Should Contain

- a clear grounded vs unsupported distinction
- artifact-backed reasoning from tool observations and verification
- a short implementation lens showing how tool contracts or verification checks support that distinction
- a revised bounded final answer

## Common Weak Patterns

- treating any tool call as grounding
- using final prose output as if it were evidence
- ignoring verification events

## Grading Nuance

The exact revised answer may differ.
What matters is whether the student narrows or qualifies claims appropriately when evidence is weak.
The implementation lens is strong when it explains why the system can verify a claim, not when it merely lists tool names.
