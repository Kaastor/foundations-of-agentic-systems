Meeting 2: “What Exactly Is the Task?”

**Why this assignment exists:** Students need to learn that a user request is not yet a control-ready task contract.

**Main reasoning-loop emphasis:** `generate -> compare -> revise -> justify`

**Expected time:** `4 hours`

**Coding level:** `bounded code-reading`, with optional workflow rerun

## Brief

Use the ambiguous literature-review request to do three things:

1. produce or obtain two plausible task-spec candidates
2. compare them against the committed course task specs
3. revise one bounded final task specification
4. inspect one small implementation surface that shows how a request becomes a validated task contract

Your final report must defend:

- which fields are essential for control
- why one candidate is stronger
- what ambiguity remains and how it should be handled

## Submission Checklist

- include at least two candidate task-spec directions
- compare against at least two committed task specs
- discuss goals, constraints, success, and stop or handoff
- inspect at least one named source module and explain which fields it makes operationally necessary
- provide one revised final task contract
- state one ambiguity you would keep explicit rather than guessing away
