# Shared Rubric

Use this rubric across all six assignments.

Score each criterion `0`, `1`, or `2`.
Default total:

- `12 points`

## 1. Task Understanding

### `2`
The report answers the right question, preserves the assignment boundary, and uses the meeting concepts correctly.

### `1`
The report is partly on target but blurs the task, overreaches, or mixes in adjacent topics.

### `0`
The report misunderstands the task or answers a different question.

## 2. Evidence Quality

### `2`
The report cites concrete book or repo artifacts precisely and uses them correctly.

### `1`
The report includes some relevant evidence, but it is thin, vague, weakly connected, or partially misused.

### `0`
The report relies mainly on opinion, paraphrase, or unsupported claims.

## 3. Comparison And Critique

### `2`
The report compares plausible options or claims fairly and clearly identifies what is correct, unsupported, or weak.

### `1`
The report attempts comparison or critique, but the analysis is shallow, one-sided, or weakly justified.

### `0`
The report largely accepts one answer at face value and shows no serious critique.

## 4. Implementation Lens

### `2`
The report identifies a relevant source module or implementation surface and explains clearly how it accounts for the observed artifact, behavior, or boundary.

### `1`
The report mentions code, but the connection to the assignment evidence is shallow, vague, or only partly correct.

### `0`
The report omits the implementation lens or treats code as decoration with no meaningful explanatory role.

## 5. Revision Or Bounded Selection

### `2`
The report makes a disciplined revision or a defended bounded choice based on the critique.

### `1`
The report includes a revision or choice, but it is weakly tied to the evidence.

### `0`
The report does not revise, narrow, or defend a meaningful choice.

## 6. Final Justification And Uncertainty

### `2`
The final judgment is clear, bounded, and appropriately qualified. Remaining uncertainty is handled honestly.

### `1`
The conclusion is partly justified but too broad, too confident, or weakly bounded.

### `0`
The report ends with unsupported confidence or no meaningful final judgment.

## Strong Submission Signals

- direct use of course artifacts
- a short but meaningful implementation lens
- for code-change assignments, a clear before/after comparison tied to one exact intervention
- clear option comparison
- explicit critique of an initial AI answer
- bounded final recommendation
- honest uncertainty without collapse into vagueness

## Weak But Polished Submission Signals

- smooth prose with few artifact references
- many general AI claims, little course-specific evidence
- superficial mention of source code with no explanatory use
- for code-change assignments, a claimed intervention with no visible consequence in the evidence
- “balanced” writing that never chooses
- superficial mention of critique with no actual revision

## Teacher Spot-Check Triggers

Escalate for human review when:

- evidence references are missing or fabricated
- the prose is very strong but the artifact use is suspiciously thin
- the report contradicts the cited artifact
- the report appears off-scope for the meeting
- the agent grader is uncertain between two adjacent scores
