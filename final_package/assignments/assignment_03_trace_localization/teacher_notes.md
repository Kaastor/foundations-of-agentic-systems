# Teacher Notes

## Why It Is Here

This assignment forces trace literacy before students move into tools, memory, and planning tradeoffs.

## What A Strong Report Should Contain

- a genuine comparison among at least two candidate evidence points
- causal use of both trace and state
- a short implementation lens explaining why the system records those evidence channels separately
- explicit connection to stop or handoff logic

## Common Weak Patterns

- narrating the whole run without localizing evidence
- naming the final handoff artifact but not the earlier causal signal
- confusing trace with memory

## Grading Nuance

Different students may choose adjacent steps.
The main question is whether they defend the “earliest decisive” criterion clearly and honestly.
For the implementation lens, do not require long code commentary; a correct structural explanation is enough.
